Fancy Return to Old School Trees

Jul 18th, 2014

Been 4-5 years since I started developing Ruby on Rails and I just don’t remember using tree views. Used them a lot in .NET development but somehow did not in Rails. This reminds me the KISS principle actually. I guess Bootstrap forms and components make us present collective data in a simple manner so we don’t tend to use treeviews to present things “all-in-one”. And for the case I will explain now, I didn’t use a treeview at first actually. I used select2. I had to make a company official to select the services they offer to customers. Services had 3-level categories and the lowest level had multiple variations. When I think of the word “select” in Rails, I instantly remember my friend Select2. I feed the JSON, Select2 presents and makes the user select things. However in this case user had to select services from ever growing thousand of services. He had to view all categories at once (obviously the top ones first), he had to filter them and for sure he had to select multiples of services. Created the JSON data at first and remotely fed it to Select2. User could see the categories, filter them and select. Things seemed right first however there was a catch. Think of google. You want to search for a wikipedia article but just can’t it with the keywords you write. Then you go to wikipedia, find a related category and you reach to the article you find by looking at its parent or child categories. This was the case for selecting services. When using select2 user could filter the services he is giving and would deinitely reach to them, however he would never see the “map” of those thousands of categories and for sure he would not include the exact services he is offering. find, I knew this could undermine the user experience so I returned to my old school friend treeview.

I had to find a treeview component to use with Rails with JSON, checkbox and filtering functionality. First checked out jstree however it was useless since clicking a node would check all the child nodes. Then bumped to Fancy tree which was exceptionally well-done.
